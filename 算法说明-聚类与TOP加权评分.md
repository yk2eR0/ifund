# 筛选任务：基金聚类 + TOP 加权评分算法

> 适用范围：`/api/screens/<id>/cluster`、`/position`、`/portfolio-analysis` 三个接口背后的完整算法链路。
> 涉及代码：`routes/screens/_scoring.py`、`_clustering.py`、`_data.py`、`routes_analysis.py`、`lib/position_math.py`。

## 0. 整体链路

一个「筛选任务（screen）」持有一组基金代码 `fund_codes`。从这组基金到「可执行的组合配置」分三层，每层各有一套打分：

```
fund_codes
   │
   ① 长期优质度评分（_score_funds）          每只基金 → 综合得分 ∈ 实数, 越高越好
   │   维度：排名一致性 / 熊市抗跌 / 风险调整收益 / 规模 / 年限
   │   产物：fund_scores 表（全市场预计算）、综合排名、质量分级
   │
   ② 行业暴露聚类（cluster_funds）            一组基金 → 若干「簇」(行业主题)
   │   方法：持仓行业/个股分布 → Bray-Curtis 相似度 → Ward 层次聚类
   │   产物：每簇 TOP10 基金（按①的综合得分排序）+ 簇行业标签 + best_fund
   │
   ③ 簇级仓位权重（compute_cluster_weights）  每簇 best_fund → 目标权重, ∑=100%
       方法：景气度(prosperity) 四因子 + 乖离度(deviation) → base×因子 → cap 迭代
       产物：可直接下单的组合权重 + 组合回测指标
```

关键点：**①评的是「基金本身长期好不好」（慢变量，全市场预计算并缓存）；③评的是「这个赛道现在该多配还是少配」（快变量，实时按净值算）。** ②只负责把同质基金归并到一起、避免组合在同一赛道上重复下注。

## 1. 长期优质度综合评分（`_score_funds`）

### 1.1 设计哲学：下限思维

核心信念写在文件头注释里——**「一个年年排前 1/3 的基金，比『前年 5% + 去年后 80%』的更优质」**。即不追求某一年的爆发，而追求穿越周期的稳定性，最差的那一年才是真实水平。

### 1.2 五个维度与权重

| 维度 | 权重 | 含义 | 数据来源 |
|---|---|---|---|
| 跨周期排名一致性 | **0.50** | 2021–2025 每年同类排名百分位，稳定且最差年份靠前 | `{year}_同类排名` |
| 长期风险调整收益 | **0.20** | 较同类风险收益比（3y/5y）+ 抗风险波动（3y） | `较同类风险收益比_近3/5年`、`较同类抗风险波动_近3年` |
| 熊市下行保护 | **0.15** | 2022 大熊市的区间收益 + 最大回撤 | `2022_区间收益`、`2022_最大回撤` |
| 规模合理性 | **0.10** | 2–5 亿最优，过大/过小递减 | `最新规模_万元` |
| 基金年限 | **0.05** | 成立越久、穿越周期越多 | `成立时间` |

> 注意：注释顶部写的是「一致性 40% / 熊市 25% / 风险 20%」，那是早期版本；**实际生效的是代码常量**（`W_ANNUAL_CONSISTENCY=0.50` 等，`_scoring.py:181-185`）。一致性提到 0.50、熊市降到 0.15，是为了「让同类排名天然承担板块相对性，降低对科技板块的偏见」。

最终综合得分：

```
_score = 0.50·一致性 + 0.20·风险调整 + 0.15·熊市 + 0.10·规模 + 0.05·年限
```

### 1.3 维度①：跨周期排名一致性（核心，0.50）

对每一年 `y∈{2021..2025}`，把同类排名换算成百分位 `pct = rank/total×100`（**0=最好，100=最差**），再按梯度打分（`_year_grade`）：

| 百分位 | 含义 | 年度分 |
|---|---|---|
| < 20% | 前 1/5，优秀 | 1.00 |
| < 33% | 前 1/3，良好 | 0.75 |
| < 50% | 前一半，及格 | 0.50 |
| < 67% | 后 1/3，偏弱 | 0.25 |
| ≥ 67% | 末段，差 | 0.00 |

然后把「均值」和「最差年（下限）」加权——**下限占 60%**，这是「下限思维」的数学落地：

```
一致性得分 = 0.4·mean(年度分) + 0.6·min(年度分)
```

- 数据要求：至少 3 年有数据（`MIN_YEARS_FOR_CONSISTENCY=3`），否则该维度记 0。
- 举例：年年前 1/3（0.75×5）得 `0.4·0.75 + 0.6·0.75 = 0.75`；而「四年优秀+一年垫底」(1,1,1,1,0) 得 `0.4·0.8 + 0.6·0 = 0.32`，被显著压低。

### 1.4 维度②：熊市下行保护（0.15）

取 2022 年的「区间收益」和「最大回撤」（回撤存储为正数，打分前取负 `dd_neg = -|dd|`），在**当前池子内**做 z-score 标准化后等权平均：

```
熊市得分 = 0.5·z(2022收益) + 0.5·z(2022回撤_负)
```

### 1.5 维度③：长期风险调整收益（0.20）

三个「较同类」指标在池内 z-score 后加权：

```
风险调整得分 = 0.4·z(风险收益比_3y) + 0.3·z(风险收益比_5y) + 0.3·z(抗风险波动_3y)
```

### 1.6 维度④：规模合理性（0.10）

分段函数 `_size_score`（单位：亿元）：

| 规模区间 | 得分 |
|---|---|
| < 1 或 > 30 亿 | 0（太迷你/太臃肿，直接出局） |
| 1–2 亿 | 0.5 → 1.0 线性爬升 |
| **2–5 亿** | **1.0（最优）** |
| 5–10 亿 | 1.0 → 0.75 递减 |
| 10–20 亿 | 0.75 → 0.33 递减 |
| 20–30 亿 | 0.33 → 0 递减 |

### 1.7 维度⑤：基金年限（0.05）

`age = (now - 成立时间)/365.25`，封顶 10 年：`age_score = min(1.0, age/10)`。

### 1.8 z-score 归一化细节（`_pool_z_scores`）

- 用**中位数**而非均值做中心（抗极端值）：`z = (v - median) / std`。
- 有效样本 < 3 或 std=0 → 全部记 0（退化保护）。
- z 值**截断到 [-2, 2]**，防止个别离群基金主导该维度。
- NaN → 0。

### 1.9 质量分级（前端展示）

按综合得分映射成档位（`_quality_tier`）：≥0.6 优秀 / ≥0.4 良好 / ≥0.2 一般 / 否则待观察。

### 1.10 预计算与缓存

- `rebuild_all_quality_scores()` 对**全市场股票型基金**批量评分，写入 `fund_scores` 表。
- 筛选任务里通过 `enrich_funds()`（`_data.py:54`）**直接读预计算分**（`load_scores_map`），不重新评分——保证全市场口径一致、排名可比。
- **质量过滤门槛** `QUALITY_SCORE_THRESHOLD = 0.5`（`_data.py:13`）：低于 0.5 的基金在建池阶段被 `_apply_quality_filter` 剔除（0.5 比 0.6 更包容消费/医药等熊市受损但优质的板块）。

## 2. 行业暴露聚类（`cluster_funds`）

目的：把持仓结构相似（同赛道/同选股风格）的基金归到一簇，避免组合在同一方向上重复下注。**输入** `fund_codes` + ①算出的 `scored_records`；**输出**按景气优先级排序的簇列表，每簇含 TOP10 基金。

前置：基金数 < 3 直接返回 `None`；无持仓数据返回 `None`。

### 2.1 个股 → 行业标签派生（`derive_label`，多级兜底）

每只持仓股票要打一个「行业主题标签」，优先级从细到粗：

1. **申万二级**（`sw_code_map`）：`sw_l2 → sw_l1 → 其他`（最细，最优先）。
2. **主题映射**（`themes`，覆盖 A 股 + 港股）：`sub_theme`（经 `THEME_SUB_TO_MAJOR` 归并）→ `etf_theme`。
3. **GB 国标行业**（`stock_industry` source=gb，粗粒度）+ 港股映射 `CLUSTER_HK_STOCK_INDUSTRY`，再过 `CLUSTER_*_TO_MAJOR` 归并到大类。
4. 都没有 → `其他`。

缺失的 A 股（6 位数字代码）会**按需**通过东方财富 `stock_individual_info_em` 在线补行业（`_ensure_stock_industry`），并用 `_ATTEMPTED_EM_CODES` 去重，避免每次聚类都打 AKShare 接口。

### 2.2 两个特征矩阵

- **行业级矩阵** `matrix`：`fund × industry_label`，值 = 该行业占净值比例之和。保留主题差异。
- **个股级矩阵** `matrix_stk`：`fund × 股票代码`，值 = 个股占净值比例。区分「同主题但选股不同」。注意只有 top10 持仓（约 35% NAV），但用于**相对比较**仍稳健。

全零行（无持仓）的基金被剔除，避免单独成簇。

### 2.3 Bray-Curtis 相似度（`_bray_curtis_sim`）

先**行内归一化为概率分布**（除以行和），消除不同总仓位（top10 占比 35% vs 80%）的影响；再算：

```
sim(a,b) = 2·Σ min(aᵢ,bᵢ) / (Σaᵢ + Σbᵢ)      ∈ [0,1]   完全相同=1, 完全不重叠=0
```

归一化后 `Σa+Σb=2`，所以 `sim = Σ min(aᵢ,bᵢ)`，省一步除法。

### 2.4 加权融合 → 距离

```
sim     = 0.6·sim_个股 + 0.4·sim_行业        # 个股权重高(区分同主题不同选股)
sim_signed = 2·sim - 1                       # [0,1] → [-1,1]，放大不重叠的距离
dist    = 1 - sim_signed                     # ∈ [0,2]，与 Pearson 距离同量纲
```

`W_STK=0.6, W_IND=0.4`。映射到 [-1,1] 是为了让「完全不重叠」对应负相关，从而在层次聚类里把不同赛道拉得更开。

### 2.5 Ward 层次聚类 + 动态簇数

```
condensed = squareform(dist)
Z = linkage(condensed, method='ward')
target_clusters = clip(n // 10, 8, 40)        # 池大自然分得多, 池小强制合并
labels = fcluster(Z, t=target_clusters, criterion='maxclust')
```

簇数随池子规模自适应：约 `n/10`，下限 8（避免太粗）、上限 40（避免每人一簇）。

### 2.6 簇标签（行业理由 + 特色重仓股 + 唯一前缀）

每簇生成一个可读标签，三段拼成：

1. **行业理由**：簇内行业均值 top3，如 `电子(28.3%); 电力设备(15.1%); ...`。
2. **特色重仓股**：簇内个股均值减去**全池基线** `pool_stock_baseline`（`matrix_stk.mean()`），取超配最多的 top3 个股名。这一步专门剔除「全池都拿」的明星股（新易盛/中际旭创/宁德时代等），凸显该簇真正的差异化持仓。
3. **唯一前缀** `[top1+top2...]`：从 (top1,top2,top3) 行业短名里取**最短的、能唯一区分本簇**的子序列做前缀，避免多个簇标签开头雷同。

最终形如：`[电子+电力设备] 电子(28.3%); 电力设备(15.1%); ... | 特色 立讯精密/沪电股份`。

### 2.7 簇排序 + 簇内 TOP10（与①衔接）

- 每簇取**簇内①综合得分最高**的基金分数作为该簇代表分 `best_per_cluster`。
- 按簇代表分降序给簇重新编号（景气/质量优先的簇排前面）。
- 每簇内基金按综合得分降序，**只保留 TOP10**（`member_recs[:10]`）。
- `best_fund_code` = 簇内得分第一名，作为③仓位计算的代表基金。

> 这就是「TOP 加权」的第一重含义：**簇用簇内最优基金的分数来代表自己**，组合最终只在每簇买 best_fund。

## 3. 簇级仓位权重（`compute_cluster_weights` / `position_math.py`）

输入 `{cluster_id: best_fund_code}` + 全量 NAV；输出每簇目标权重（∑=100%）。**这一层完全基于净值时间序列实时计算，与①的长期分无关**——衡量的是「赛道当下的景气与位置」。迁移自 fund-selection 的 `02_position_sizing.py`。

核心公式：

```
簇权重 ∝ base(1/N) × prosperity_factor × deviation_factor
        → cap 到 [3%, 25%] → 迭代再分配超额 → 归一化 ∑=100%
```

### 3.1 景气度 prosperity（四因子，0–100）

`PROSPERITY_WEIGHTS`：

| 因子 | 权重 | 计算（均以 best_fund 净值为单基金代理） |
|---|---|---|
| **momentum 动量** | 0.35 | 1m/3m/6m 收益按 `0.5/0.3/0.2` 加权，再 **min-max** 归一化到 [0,100] |
| **risk_adj 风险调整** | 0.25 | 近 60 日 `mean(日增长率)/std×√252`（Sharpe-like），**rank** 归一化 |
| **breadth 广度** | 0.25 | 净值站上 MA20/MA60 的程度 `0.6·above20 + 0.4·above60`，再加乖离深度 bonus（≤10） |
| **consistency 一致性** | 0.15 | 从最近往前数**连续正收益月数 / 总月数**（月=20 交易日） |

```
prosperity = 0.35·momentum + 0.25·risk_adj + 0.25·breadth + 0.15·consistency
```

数据不足时各因子回退到中性 50。归一化方式刻意混用：动量用 min-max（保留差距大小），风险调整用 rank（抗极端值）。

### 3.2 乖离度 deviation（择时，`_compute_price_deviation`）

当前净值相对均线的乖离：`d20 = (nav-MA20)/MA20×100`，`d60` 同理，`combined = 0.6·d20 + 0.4·d60`。用于「高景气但已涨多 → 适度收手；高景气且回踩 → 加码」的择时微调。

### 3.3 目标权重合成（`_compute_target_weights`）

`base = 1/N`，两个调节因子相乘：

**景气因子**（按池内 z-score 拉开，封顶 ±50%）：
```
pros_factor = clip( 1 + 0.30·(pros - mean_pros)/std_pros , 0.5, 1.5 )
```

**乖离因子**（分景气强弱、方向，`tanh` 软饱和，封顶 ±40%）：
- `|dev| ≤ 3%`：中性 `1.0`。
- **景气强（pros≥50）**：无论涨跌都视为「趋势确立」→ 适度**加码** `1 + 0.25·tanh(|dev|/5)`。
- **景气弱（pros<50）**：涨多了（dev>3）**减码** `1 - 0.25·tanh(dev/5)`；跌深了（dev<-3）小幅减 `1 - 0.125·tanh(...)`（弱势不抄底）。

```
raw_weight = base × pros_factor × dev_factor
```

### 3.4 Cap + 迭代再分配

先按 raw 归一化，再把权重 cap 到 **[min 3%, max 25%]**，超出/不足的部分**按未触顶簇的现有权重比例迭代摊派**（最多 10 轮，`|excess|<1e-6` 提前收敛）：

```
for _ in range(10):
    把 <3% 上拉到 3%、>25% 压到 25%，累计 excess
    剩余未触顶簇按各自权重占比让出/吸收 excess
```

保证：单簇不会过度集中（≤25%）也不会形同虚设（≥3%），最终 ∑≈100%。

> 这是「TOP 加权」的第二重含义：**每簇只配 best_fund 一只，簇间按景气+位置加权**，得到最终可下单的组合。

### 3.5 退化保护

`nav_df` 为空时全簇等权 `1/N`、prosperity=50、各因子=1.0，前端仍可正常渲染。

## 4. 组合回测（`/portfolio-analysis`）

用③的簇权重 + 各簇 best_fund 净值，在公共日期区间（1y/3y/all）归一化（起点=1）后加权合成组合净值，计算：年化收益、年化波动、Sharpe（`ann_ret/ann_vol`，rf=0）、最大回撤、Calmar（`ann_ret/|max_dd|`）。要求公共净值日 ≥ 20 天。

## 6. 已知约束与坑

- **硬编码 YTD 日期** `2026-01-01`（`fund.py`/`screens.py`），每年初需手改。
- **`_scoring.py` 头注释权重已过时**，以代码常量为准。
- **个股矩阵仅 top10 持仓**（~35% NAV），聚类基于不完整持仓，相对比较稳健但非全持仓。
- **离线批处理副本**：`scripts/compute_weighted_score.py` 与 `_scoring._score_funds()` 是近似双胞胎（offline vs live），改评分逻辑要两边同步。
- **景气度均为单基金代理**：breadth/consistency 本应是「簇内成分股广度」，这里用 best_fund 单只净值近似，是工程简化。
- A 股行业在线补全依赖 AKShare，受限流影响；`_ATTEMPTED_EM_CODES` 仅进程内去重，重启后会重试。
